# 楽天市場 広告入札単価調整アプリ 仕様書

## 1. システム構成

### 1.1 技術構成（案）

| レイヤー | 技術 | 備考 |
|---------|------|------|
| フロントエンド | HTML / CSS / JavaScript | ブラウザ上で完結（SPA） |
| CSVパース | ブラウザ側で処理 | サーバー不要 |
| エンコーディング変換 | Encoding API / ライブラリ | Shift-JIS ⇔ UTF-8 |

**方針:** サーバーレス構成。CSVの読み込み・計算・出力をすべてブラウザ上で行う。データをサーバーに送信しないため、セキュリティリスクが低い。

## 3. 処理仕様

### 3.1 CSVファイル読み込み

```
入力: Shift-JIS エンコードのCSVファイル
処理:
  1. FileReader API でファイルを ArrayBuffer として読み込む
  2. TextDecoder('shift-jis') で UTF-8 文字列に変換
  3. 1〜6行目（メタデータ）をスキップ
  4. 7行目をカラムヘッダーとしてパース
  5. 8行目以降をデータ行としてパース
  6. 各フィールドのダブルクォート囲みを除去
  7. 数値フィールドは適切な型に変換
出力: パース済みデータ配列
```

### 3.2 キーワードデータ集計

```
入力: keyword.csv のパース済みデータ
処理:
  1. 商品管理番号をキーとしてグルーピング
  2. 各グループで以下を合計:
     - クリック数(合計)
     - 実績額(合計)
     - 売上金額(合計12時間)
出力: { 商品管理番号 → { clicks, cost, sales } } のMap
```

### 3.3 データ差し引き・ROAS再計算

```
入力: item.csv データ, keyword集計Map
処理:
  各商品について:
    kwData = keyword集計Map[商品管理番号] または { clicks:0, cost:0, sales:0 }
    adjustedClicks = item.クリック数 - kwData.clicks
    adjustedCost   = item.実績額 - kwData.cost
    adjustedSales  = item.売上金額12h - kwData.sales
    adjustedROAS   = adjustedCost > 0 ? (adjustedSales / adjustedCost) * 100 : 0
出力: 調整後データ配列
```

### 3.4 入札単価調整

```
入力: 調整後データ配列, ルール設定
処理:
  各商品について:
    1. adjustedROAS と adjustedClicks からルールをマッチ
    2. マッチしたルールの調整値を取得
    3. newBid = currentBid + adjustment
    4. if newBid < minBid: newBid = minBid
    5. isAlert = (newBid == minBid) && (adjustedROAS <= alertThreshold)
出力: 結果データ配列
```

### 3.5 CSV出力

```
入力: 結果データ配列
処理:
  1. ヘッダー行を生成: "コントロールカラム,商品管理番号,商品CPC"
  2. 各商品のデータ行を生成: ",{商品管理番号},{新入札単価}"
  3. UTF-8文字列 → Shift-JIS に変換
  4. Blob を生成し、ダウンロードリンクを作成
出力: Shift-JIS エンコードのCSVファイル（ブラウザダウンロード）
```

## 4. ルール設定の詳細仕様

### 4.1 ルールデータ構造

```javascript
// ルール1件の構造
{
  id: 1,                    // ルール番号（表示・優先度用）
  roasMin: 0,               // ROAS下限（null = 下限なし）
  roasMax: 0,               // ROAS上限（null = 上限なし）
  roasMinInclusive: true,   // 下限を含むか
  roasMaxInclusive: true,   // 上限を含むか
  clickCondition: 'lt',     // クリック数条件（lt, gte, null=条件なし）
  clickThreshold: 30,       // クリック数閾値
  adjustment: -1            // 調整値（円）
}
```

### 4.2 デフォルトルール定義

```javascript
const DEFAULT_RULES = [
  { id:1, roasMin:0, roasMax:0, roasMinInclusive:true, roasMaxInclusive:true,
    clickCondition:'lt', clickThreshold:30, adjustment:-1 },
  { id:2, roasMin:0, roasMax:0, roasMinInclusive:true, roasMaxInclusive:true,
    clickCondition:'gte', clickThreshold:30, adjustment:-3 },
  { id:3, roasMin:0, roasMax:250, roasMinInclusive:false, roasMaxInclusive:false,
    clickCondition:null, clickThreshold:null, adjustment:-1 },
  { id:4, roasMin:250, roasMax:400, roasMinInclusive:true, roasMaxInclusive:true,
    clickCondition:null, clickThreshold:null, adjustment:0 },
  { id:5, roasMin:400, roasMax:null, roasMinInclusive:false, roasMaxInclusive:false,
    clickCondition:null, clickThreshold:null, adjustment:3 },
];

const DEFAULT_MIN_BID = 20;
const DEFAULT_ALERT_ROAS = 150;
```

### 4.3 ルールマッチングロジック

ルールは上から順に評価し、**最初にマッチ**したルールを適用する。

```
function matchRule(roas, clicks, rules):
  for rule in rules:
    if matchROAS(roas, rule) and matchClicks(clicks, rule):
      return rule
  return null  // マッチなし → 調整値0

function matchROAS(roas, rule):
  lower = rule.roasMinInclusive ? (roas >= rule.roasMin) : (roas > rule.roasMin)
  upper = rule.roasMax == null ? true :
          rule.roasMaxInclusive ? (roas <= rule.roasMax) : (roas < rule.roasMax)
  return lower and upper

function matchClicks(clicks, rule):
  if rule.clickCondition == null: return true
  if rule.clickCondition == 'lt': return clicks < rule.clickThreshold
  if rule.clickCondition == 'gte': return clicks >= rule.clickThreshold
```

## 5. エラーハンドリング

### 5.1 ファイル関連エラー

| エラー | メッセージ | 対処 |
|--------|----------|------|
| ファイル未選択 | 「商品別レポートとキーワード別レポートの両方を選択してください」 | 実行ボタン無効化 |
| CSV以外のファイル | 「CSVファイルを選択してください」 | 処理中止 |
| ヘッダー不正 | 「ファイルのフォーマットが正しくありません。楽天市場からダウンロードしたCSVか確認してください」 | 処理中止 |
| データ0件 | 「データが見つかりません」 | 処理中止 |

### 5.2 データ関連エラー

| エラー | メッセージ | 対処 |
|--------|----------|------|
| 数値変換失敗 | 「行XX: 数値の変換に失敗しました」 | 該当行スキップ、警告表示 |
| マイナス値発生 | 「行XX: 差し引き後の値がマイナスになりました」 | 0として扱い、警告表示 |

### 5.3 ルール設定エラー

| エラー | メッセージ | 対処 |
|--------|----------|------|
| 閾値が数値でない | 「数値を入力してください」 | 入力フィールドをハイライト |
| 下限値が0以下 | 「下限値は1以上で設定してください」 | 入力フィールドをハイライト |
| ルール0件 | 「ルールを1つ以上設定してください」 | 実行ボタン無効化 |

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンス | CSV処理完了まで3秒以内（100商品規模） |
| ブラウザ対応 | Chrome, Edge, Firefox 最新版 |
| データ保持 | サーバーへのデータ送信なし（クライアント完結） |
| ルール保持 | ブラウザのlocalStorageにルール設定を保存（次回起動時に復元） |
| 画面幅 | PC向け（最小幅1024px） |
