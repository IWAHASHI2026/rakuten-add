# 楽天市場 広告入札単価調整アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
楽天市場の広告パフォーマンスデータ（ROAS・クリック数）に基づき、商品ごとの入札単価を自動調整するWebアプリケーションを開発する。手作業による単価調整の工数を削減し、広告運用の効率化を図る。

### 1.2 対象ユーザー
楽天市場の広告運用担当者

### 1.3 対象店舗
naireya-bekkan（楽天市場出店店舗）

### 1.4 アプリ形態
Webアプリケーション（ブラウザ上で動作）

### 1.5 実行頻度
任意のタイミングでユーザーが手動実行

---

## 2. 入力データ仕様

### 2.1 ファイルアップロード方式
- ブラウザ上のファイル選択UIから2つのCSVファイルをアップロードする
- ファイル名は任意（楽天管理画面からダウンロードしたファイル名のまま使用可能）

### 2.2 商品別レポート（item.csv相当）
楽天市場の広告管理画面から「集計単位: 商品別」でダウンロードしたCSVファイル。

| 項目 | 説明 |
|------|------|
| エンコーディング | Shift-JIS |
| ヘッダー行 | 1〜6行目はメタデータ（実行日時、検索条件等） |
| カラムヘッダー | 7行目 |
| データ開始行 | 8行目以降 |

**主要カラム:**

| カラム名 | 型 | 説明 | 用途 |
|----------|------|------|------|
| コントロールカラム | 文字列 | 楽天側の制御カラム（通常空） | 出力時に使用 |
| 商品ページURL | 文字列 | 商品のURL | 参照・表示用 |
| 商品管理番号 | 文字列 | 商品を一意に識別するID | **キー項目** |
| 入札単価 | 整数 | 現在の商品CPC入札単価（円） | **調整対象** |
| クリック数(合計) | 整数 | 期間中の合計クリック数 | ルール判定に使用 |
| 実績額(合計) | 整数 | 期間中の広告費合計（円） | ROAS再計算に使用 |
| 売上金額(合計12時間) | 整数 | 12時間以内の売上金額（円） | ROAS再計算に使用 |
| ROAS(合計12時間)(%) | 小数 | 12時間ROASの値 | ※キーワード分含む合計値 |

### 2.3 キーワード別レポート（keyword.csv相当）
楽天市場の広告管理画面から「集計単位: キーワード」でダウンロードしたCSVファイル。

| 項目 | 説明 |
|------|------|
| エンコーディング | Shift-JIS |
| ヘッダー行 | 1〜6行目はメタデータ |
| カラムヘッダー | 7行目 |
| データ開始行 | 8行目以降 |

**主要カラム:**

| カラム名 | 型 | 説明 |
|----------|------|------|
| 商品管理番号 | 文字列 | item.csvと紐付けるキー |
| キーワード | 文字列 | 設定されたキーワード |
| キーワードCPC | 整数 | キーワードごとのCPC（通常空） |
| クリック数(合計) | 整数 | キーワード経由の合計クリック数 |
| 実績額(合計) | 整数 | キーワード経由の広告費合計 |
| 売上金額(合計12時間) | 整数 | キーワード経由の12時間以内売上 |

**特記事項:** 1つの商品管理番号に対して複数のキーワード行が存在する（1対多の関係）。

### 2.4 アップロード用CSV出力フォーマット

| カラム名 | 型 | 説明 |
|----------|------|------|
| コントロールカラム | 文字列 | 空文字 |
| 商品管理番号 | 文字列 | 対象商品の管理番号 |
| 商品CPC | 整数 | **調整後の入札単価** |

**出力エンコーディング:** Shift-JIS（楽天市場アップロード仕様に準拠）

## 3. データ前処理仕様

### 3.1 キーワードデータの差し引き

item.csv（商品別レポート）の数値には、keyword.csv（キーワード別レポート）の数値が含まれている。商品CPC（商品レベル広告）のパフォーマンスのみで判定するため、キーワード分を差し引く。

**処理手順:**

1. keyword.csvの各商品管理番号ごとに、以下を集計する:
   - クリック数(合計)の合計
   - 実績額(合計)の合計
   - 売上金額(合計12時間)の合計

2. item.csvの各商品から、対応するキーワード集計値を差し引く:
   - **調整後クリック数** = item.クリック数(合計) - keyword集計.クリック数(合計)
   - **調整後実績額** = item.実績額(合計) - keyword集計.実績額(合計)
   - **調整後売上金額** = item.売上金額(合計12時間) - keyword集計.売上金額(合計12時間)

3. 調整後のROASを再計算する:
   - **調整後ROAS(%)** = (調整後売上金額 / 調整後実績額) × 100
   - 調整後実績額が0の場合: ROAS = 0 とする

---

## 4. 入札単価調整ルール

### 4.1 デフォルト調整ルール

調整後のROAS(合計12時間)(%)とクリック数(合計)に基づき、現在の入札単価に対して以下の調整を行う。

| # | 条件 | 調整値 |
|---|------|--------|
| 1 | ROAS = 0 かつ クリック数 < 30 | **-1円** |
| 2 | ROAS = 0 かつ クリック数 >= 30 | **-3円** |
| 3 | 0 < ROAS < 250 | **-1円** |
| 4 | 250 <= ROAS <= 400 | **0円（据え置き）** |
| 5 | ROAS > 400 | **+3円** |

**計算式:**
```
新入札単価 = 現在の入札単価 + 調整値
```

### 4.2 ルールのカスタマイズ機能

ユーザーが画面上で以下の項目を任意に変更できること:

- **ROASの閾値**: 各条件のROAS境界値（0, 250, 400 等）
- **クリック数の閾値**: ROAS=0時のクリック数境界値（30）
- **調整値**: 各条件に対応する入札単価の増減値（-1, -3, 0, +3 等）
- **入札単価の下限値**: デフォルト20円
- **アラート条件のROAS閾値**: デフォルト150

### 4.3 入札単価の下限値

- 入札単価の下限値はデフォルト **20円**（カスタマイズ可能）
- 調整後の入札単価が下限値を下回る場合、**下限値に設定**する

### 4.4 境界値の定義

| 条件の記載 | 解釈 |
|-----------|------|
| ROAS = 0 | ROAS が 0.00 |
| 0 < ROAS < 250 | 0より大きく250未満 |
| 250 <= ROAS <= 400 | 250以上400以下 |
| ROAS > 400 | 400より大きい |

## 5. アラート機能

### 5.1 アラート条件

以下の条件を**両方**満たす商品をアラート表示する:

- 入札単価（調整後）が **下限値**（デフォルト20円）
- 調整後ROAS(合計12時間)(%) が **150以下**（カスタマイズ可能）

### 5.2 アラートの意図

入札単価が下限に達しており、かつROASが低い商品は、広告効果が低い可能性があるため、**広告停止や商品の見直し**を検討すべき対象として注意を促す。

## 6. 出力仕様

### 6.1 アップロード用CSV

- ファイル形式: CSV（カンマ区切り）
- エンコーディング: Shift-JIS
- ヘッダー行: あり（コントロールカラム,商品管理番号,商品CPC）
- **出力対象: 全商品（調整値0円の商品も含む）**
- ダウンロード: ブラウザからファイルダウンロード

### 6.2 結果一覧表示

処理結果を画面上にテーブルで一覧表示する。

表示項目:
- 商品管理番号
- 商品ページURL
- 現在の入札単価
- 調整後ROAS(合計12時間)(%)
- 調整後クリック数(合計)
- 調整値
- 新入札単価
- アラート有無

### 6.3 アラート表示

アラート対象商品を**視覚的に強調表示**する（背景色変更、警告アイコン等）。

## 7. 処理フロー

```
[開始]
  │
  ├─ 1. ユーザーがルール設定を確認・変更（任意）
  │
  ├─ 2. 商品別レポートCSVをファイル選択UIからアップロード
  ├─ 3. キーワード別レポートCSVをファイル選択UIからアップロード
  │
  ├─ 4. CSVファイル読み込み（Shift-JIS → 内部処理用に変換）
  │
  ├─ 5. キーワードデータを商品管理番号ごとに集計
  │     （クリック数、実績額、売上金額の合計）
  │
  ├─ 6. 商品データからキーワード集計値を差し引き
  │     → 調整後クリック数、実績額、売上金額を算出
  │
  ├─ 7. 調整後ROASを再計算
  │     ROAS = (調整後売上金額 / 調整後実績額) × 100
  │
  ├─ 8. 入札単価調整ルールを適用
  │     → 調整後ROAS・クリック数から調整値を決定
  │     → 新入札単価 = 現在の入札単価 + 調整値
  │
  ├─ 9. 入札単価の下限チェック
  │     → 下限値未満の場合は下限値に設定
  │
  ├─ 10. アラートチェック
  │      → 入札単価=下限値 かつ ROAS<=アラート閾値 の商品を抽出
  │
  ├─ 11. 結果一覧を画面に表示（アラート対象は強調表示）
  │
  ├─ 12. アップロード用CSV出力（全商品）
  │      → ブラウザからダウンロード
  │
[終了]
```

## 8. 処理例

### 例1: uvchy042-mgyokoof

| 項目 | item.csv | keyword集計 | 調整後 |
|------|---------|------------|--------|
| クリック数(合計) | 298 | 3 | 295 |
| 実績額(合計) | 10,516 | 240 | 10,276 |
| 売上金額(合計12時間) | 22,740 | 0 | 22,740 |
| ROAS(合計12時間)(%) | 216.24 | - | **221.30** |

- 現在の入札単価: 35
- 調整後ROAS: 221.30（0 < ROAS < 250）→ 調整値: -1
- **新入札単価: 34**
- アラート: 対象外

### 例2: uv030-seineu

| 項目 | item.csv | keyword集計 | 調整後 |
|------|---------|------------|--------|
| クリック数(合計) | 45 | 2 | 43 |
| 実績額(合計) | 592 | 120 | 472 |
| 売上金額(合計12時間) | 1,080 | 0 | 1,080 |
| ROAS(合計12時間)(%) | 182.43 | - | **228.81** |

- 現在の入札単価: 20
- 調整後ROAS: 228.81（0 < ROAS < 250）→ 調整値: -1
- 下限チェック: 20 - 1 = 19 < 20 → **新入札単価: 20**
- アラートチェック: 入札単価=20、ROAS=228.81 > 150 → **アラート対象外**

### 例3: アラート対象の例

- 現在の入札単価: 21
- 調整後ROAS: 88.46（0 < ROAS < 250）→ 調整値: -1
- **新入札単価: 20**
- アラートチェック: 入札単価=20、ROAS=88.46 <= 150 → **アラート対象**
